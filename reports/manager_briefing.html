Method

Life is right-skewed (Weibull β ≈ 1), so all tests run on log-life or use the non-parametric Kruskal–Wallis, reporting effect size (ε² / η²), not just p. Blame is separated by a nested variance decomposition: part (design) → lot within part (batch) → within-lot residual (operation) — the standard batch-vs-within-batch split used for supplier-vs-process diagnosis in manufacturing.

Read the fine print in the meeting. “Us” is the residual after design + batch, so it also absorbs random noise — treat 54% as an upper bound. There is no true lot number or machine ID in the data yet (lot is proxied by install date), and the 2024 Q3 removal spike needs a maturation-bias check — recently-installed units still running aren’t in the data — before it’s claimed as a hard event. The direction of every finding is solid; the exact percentages are soft.
